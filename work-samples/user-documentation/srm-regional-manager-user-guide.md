### Purpose

The purpose of this guide is to help Regional Managers navigate the SRM portal and use its features to perform their daily activities.

### Prerequisites

Regional Manager access to the SRM portal.

### SRM Portal Interface

The SRM portal consists of **6** modules and a **Home** page: **Company Leads, Placement Occurrences, Targets, Calendar, People, Tasks, and Reports**.

The **Quick Actions** feature includes **Global Search**, **Favorite List**, **Global Actions**, **Guidance Center**, **Help**, **Setup**, **Notifications**, and **View Profile**.

The **Home** page includes the **Dashboard** and **Targets**.

### Quick Actions

**Global Search:** This field allows users to quickly find relevant information across the entire system by entering keywords or phrases. It provides a centralized search function that scans all available data, including records, documents, discussions, and more.

**Favorite List:** This feature allows users to bookmark and save frequently accessed items or records for quick reference. Users can add specific items, such as reports, records, dashboards, or pages, to their Favorite List with a single click.

**Global Actions:** These actions are commands or functions available from any screen or module within the system. They typically include common operations such as creating new records, sending messages, starting workflows, or performing system-wide updates.

**Guidance Center:** This center serves as a central hub for help resources, tutorials, documentation, and best practices within the system. It helps users navigate the system effectively, troubleshoot issues, and improve productivity.

**Help:** This feature gives users access to Salesforce documentation, tutorials, and support resources to help them use the system effectively.

**Notifications:** This feature alerts users to important events, updates, or actions within the system in real time. Notifications may include new messages, task assignments, upcoming events, or changes to records they follow.

**View Profile:** This feature allows users to access and manage their personal profile information, such as name, contact information, preferences, and profile picture.

#### Creating a New Company Lead

To create a **new company lead**, follow the steps below:

1. Click the **New** button.

<div align="left"><img src="../../.gitbook/assets/Unknown image (102)" alt=""></div>

The **New Company Lead** dialog box is displayed.

<div align="left"><img src="../../.gitbook/assets/Unknown image (103)" alt=""></div>

1. Select the desired **record type**, then click **Next**.

**Domestic Lead:** This lead type focuses on potential clients or partners within the same country or geographic region.

**International Lead:** This lead type focuses on potential clients or partners from foreign countries or global markets.

<div align="left"><img src="../../.gitbook/assets/Unknown image (104)" alt=""></div>

The **New Company Lead: Domestic Lead** dialog box is displayed.

1. Fill in the following fields:

| **Field**                               | **Description**                                                                                                                                                                                                                                 |
| --------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Company Name (\*)**                   | This field serves as the primary identifier for the organization or entity. It is a mandatory field where users enter the official name of the company or business entity.                                                                      |
| **CIN (\*)**                            | This field is a unique identifier assigned to registered companies in certain jurisdictions. It is a mandatory field where users input the alphanumeric code or registration number issued by the relevant regulatory authority.                |
| **Website (\*)**                        | This field allows users to enter the official website address of the company. It is a mandatory field where users provide the URL (Uniform Resource Locator) linking to the company's online presence.                                          |
| **Industry**                            | This field specifies the sector or category to which the company belongs. Users select or input the primary industry or industries in which the company operates, helping to categorize and classify its business activities.                   |
| **Primary Address**                     | This field captures the main physical location or mailing address of the company. It includes details such as street address, city, state/province, postal code, and country/territory.                                                         |
| **Primary Address (Country/Territory)** | This field specifically records the country or territory where the primary address of the company is located.                                                                                                                                   |
| **Primary address (Street)**            | This field contains the street name and number of the primary physical location of the company. Users input the street address where the company is situated.                                                                                   |
| **Primary address (City)**              | This field records the name of the city or locality associated with the primary address of the company. Users enter the city where the company's main office or location is situated.                                                           |
| **Primary address (State/Province)**    | This field specifies the state or province corresponding to the primary address of the company. Users select or input the state or province where the company's main office or location is situated.                                            |
| **Primary address (ZIP/Postal Code)**   | This field captures the postal code or ZIP code of the primary address of the company. Users enter the code associated with the area or region where the company is located, facilitating accurate mail delivery and geographic identification. |

<div align="left"><img src="../../.gitbook/assets/Unknown image (105)" alt=""></div>

1. Click **Save** to save the lead, **Save & New** to save the current lead and start a new one, or **Cancel** to discard the lead.

### Targets

The Targets module enables users to set and monitor specific goals, objectives, or key performance indicators (KPIs) for individuals, teams, or projects within the organization. Users can define target metrics, assign responsibilities, set deadlines, and track progress over time. This module supports performance management, encourages accountability, and aligns work with organizational objectives.

### Calendar

The Calendar module provides a visual representation of scheduled events, appointments, and deadlines. It allows users to view, create, edit, and manage schedules efficiently. With features such as customizable views, reminders, and integration with other modules, the Calendar module helps users stay organized, prioritize tasks, and coordinate activities effectively.
